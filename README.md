# CDM Audio Processor

CDM Audio Processor — экспериментальный realtime DSP-процессор для выравнивания субъективного баланса в кино, играх и многоканальном аудио.

Проект родился из простой раздражающей проблемы: в фильмах диалоги часто утоплены в миксе, а взрывы, музыка, LFE и окружение орут так, будто задача звукорежиссёра — разбудить соседей, но не дать услышать, что говорят персонажи. Приходится постоянно держать пульт в руке: прибавлять на речи, убавлять на экшене, потом снова прибавлять. CDM пытается автоматизировать этот процесс на уровне аудиопотока.

## Идея

Алгоритм не является обычным эквалайзером или компрессором всего сигнала. Вместо грубого изменения общей громкости он пытается разложить входной аудиоблок на смысловые компоненты:

```text
VOICE    — речь и центр микса
EFFECTS  — музыка, шумы, окружение, фронтальные и surround-эффекты
BASS     — LFE и низкочастотная энергия
```

После этого CDM оценивает, насколько активна речь, и динамически меняет баланс между этими шинами:

```text
voice_gain    ↑ когда речь маскируется
effects_gain  ↓ когда эффекты забивают голос
bass_gain     ↓ когда LFE мешает разборчивости
```

Цель — не “сделать всё одинаково громким”, а сохранить читаемость речи без полного уничтожения динамики, сцены и баса.

## Общий pipeline

Типовой путь обработки выглядит так:

```text
input audio block
→ channel layout resolve
→ dialogue candidate extraction
→ effects bus build
→ bass bus build
→ speech / masking analysis
→ adaptive gain
→ downmix / output matrix
→ limiter
→ output audio block
```

Алгоритм работает с короткими блоками samples и рассчитан на realtime path. Внутренний формат — `float32`, interleaved или приведённый к нему адаптером конкретной платформы.

Основные входные каналы:

```text
FL / FR / FC / LFE / SL / SR / BL / BR
```

Если центральный канал `FC` доступен, он используется как главный источник диалогов. Если вход только стерео, голосовая компонента оценивается через phantom center из `FL / FR`.

## Что делает DSP-ядро

CDM Core выполняет несколько стадий обработки:

```text
1. нормализация входного блока
2. определение раскладки каналов
3. построение VOICE bus
4. построение EFFECTS bus
5. построение BASS bus
6. оценка speech confidence
7. расчёт adaptive gain
8. применение gain к смысловым шинам
9. сборка L/R/SUB выхода
10. limiter / output shaping
```

Важный момент: speech detector не смешивает звук и не является отдельным audio path. Он только анализирует текущий блок и выдаёт коэффициенты управления. Сам сигнал остаётся в основных DSP-шинах.

## Зачем это нужно

CDM делает более адресную обработку:

```text
если речь активна:
  voice поднимается
  effects слегка проседают
  bass/lfe контролируется

если речи нет:
  микс возвращается ближе к исходной динамике
```

Это особенно полезно для:

```text
- фильмов с тихими диалогами и резкими экшен-сценами
- игр с громким окружением и неразборчивыми репликами
- 5.1 / 7.1 дорожек
- downmix в 2.0 / 2.1
- просмотра ночью без постоянного дергания громкости
```


DSP-ядро не должно зависеть от конкретной ОС, GUI или драйверной модели. Платформенный слой должен только привести звук к ожидаемому формату, вызвать CDM Core и вернуть обработанный блок обратно.

## Текущий статус

Сейчас проект является исследовательским прототипом.

Реализовано:

```text
- realtime DSP core
- построение VOICE / EFFECTS / BASS шин
- speech confidence estimator
- adaptive gain для речи, эффектов и баса
- downmix в 2.0 / 2.1
- output limiter
- диагностическая визуализация каналов и шин
- live audio capture для тестов
- экспериментальный bridge для Windows APO
```

Проект пока находится на стадии проверки алгоритма, бэкенда и живого поведения. Основной фокус — понять, насколько хорошо можно автоматически вытаскивать речь из сложного микса без превращения всего звука в пережатую кашу.

## Ограничения

CDM не восстанавливает то, чего нет в исходном миксе. Если речь уже убита кодеком, сведена в кашу или полностью замаскирована эффектами, алгоритм может только улучшить баланс, но не извлечь идеальный чистый голос.

Качество работы сильно зависит от входного материала:

```text
5.1 / 7.1 с отдельным FC обычно даёт лучший результат
стерео требует оценки phantom center
битстрим Dolby/DTS нужно предварительно декодировать в PCM
браузерные плееры часто отдают только FL/FR
```
